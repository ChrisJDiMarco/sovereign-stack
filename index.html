What's Inside the Guide

The complete playbook.
Every decision already made.

This isn't a theory doc — it's the exact architecture, with every config, every command, and the gotchas spelled out. Free on GitHub, it takes you from a blank VPS to a running — and properly hardened — three-agent stack.

01

VPS Selection + Bare-Metal Setup

A small VPS (Hetzner CX22 is ~$5/month). Ubuntu 24.04, NVM, PM2 for auto-restart, Caddy for automatic HTTPS — in under an hour.

02

Hermes Agent — Install + Configuration

Nous Research's open-source persistent-memory agent: small, curated memory files plus SQLite/FTS5 keyword recall across sessions, 60+ built-in tools, and a gateway that listens on Telegram. Claude-first, with OpenAI / OpenRouter / any OpenAI-compatible endpoint.

03

Paperclip Control Plane

The open-source "AI company OS." Visual org chart, a wakeup queue with budget checks, per-agent cost caps, and approval flows — the governance layer your agents need.

04

Telegram Gateway — Your Phone Interface

Wire up the Hermes gateway for Telegram. DM your CEO agent from anywhere, get a reply in about a second. No laptop required.

05

Day 1 Org Chart — 3 Agents That Work

CEO (orchestration + memory), Research (web intel on demand), Builder (Claude Code + Paperclip tickets). A proven starting lineup with system prompts and budget caps included.

06

Event-Driven Wakeups via n8n

Business signals (new filings, competitor posts, risk spikes) → webhook → an agent wakes with context. Proactive, not just scheduled.

07

Security & Isolation — Hardened by Default

Zero public ports (Tailscale), a sandboxed code-execution agent, scoped service-account keys, risk-tiered approval gates, and a prompt-injection boundary for the web-reading agent. The chapter the free tools skip — plus cost circuit-breakers, liveness supervision, safe backups, and tracing.

08

Voice Layer (Optional)

Hands-free dictation on Mac. Use macOS's built-in on-device dictation to stay fully local, or the cloud-based Vibing app (powered by Microsoft VibeVoice) for richer rewriting — with the privacy trade-off spelled out, so you choose.

09

Architecture Diagram + Honest Cost Model

Every component mapped, and the real running cost broken down line by line — about $5/mo for the VPS, ~$69/mo all-in once you add your own (capped) API usage. No hidden math.

Rent vs. Own

Managed cloud agents are great.
Until you want to own them.

Tools like Perplexity Computer are genuinely good, and if you'd rather rent, go for it — they prove the market is real. And if you'd rather self-host with a free tool like OpenClaw, you can — this guide's edge is the curated, secured path (the free tools are powerful but easy to misconfigure into a real security incident). But if you want the memory, the data, and the cost ceiling to live on hardware you control, you self-host. Here's the honest trade-off.

What matters Managed cloud Sovereign Stack
Where your memory lives Their cloud Your server (SQLite)
Where your data lives Their servers Your VPS only
Phone interface Their app Telegram DM you control
Cost model Metered credits ~$5/mo VPS + your capped keys
Cost ceiling per agent Plan limits Hard caps via Paperclip
Model / provider choice Their routing Claude-first, swap freely
Who owns it Your subscription You do — forever

Managed services are for people who want someone else to run everything — and that's a perfectly good choice. The Sovereign Stack is for the operator who refuses to rent their second brain: who wants to own the infrastructure, cap the costs, and keep the data on a box they control. The software is free and open source. This guide just saves you the weekend of figuring out how the pieces fit.
